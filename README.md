# Student_Planner_Belyaev_Selivanov

## Описание приложения

Student Planner — это мобильное приложение для студентов, позволяющее отслеживать учебные дисциплины, просматривать расписание и управлять учебным процессом. Приложение помогает организовать информацию о курсах, преподавателях и успеваемости, а также планировать учебное время.

## Реализованные экраны
- **Home** — главный экран со списком дисциплин
- **Details** — экран с подробной информацией о выбранной дисциплине
- **Profile** — экран профиля студента
- **Settings** — экран настроек приложения
- **Schedule** — экран с расписанием занятий на неделю
- **LessonDetail** — экран детальной информации о занятии

## Используемые технологии
- **Kotlin** — основной язык программирования
- **Jetpack Compose** — современный UI toolkit
- **Navigation Compose** — библиотека для навигации между экранами
- **Material 3** — дизайн-система Google

## Схема навигации между экранами
- Home → Profile → (назад) → Home
- Home → Settings → (назад) → Home
- Home → Schedule → LessonDetail → (назад) → Schedule → (назад) → Home
- Home → Details → (назад) → Home

## Контрольные вопросы

### 1. Что такое NavController и для чего он используется?
**NavController** — компонент управления навигацией. Отвечает за переходы между экранами, стек навигации (back stack) и кнопку "Назад". Создается через `rememberNavController()`, чтобы сохранять состояние при рекомпозиции UI.

### 2. Как передать параметр в маршрут навигации?
**Процесс:**
1. Определение: `"details/{subjectId}"`
2. Переход: `navController.navigate("details/123")`
3. Извлечение: `backStackEntry.arguments?.getString("subjectId")`

**Обязательные параметры** всегда присутствуют в маршруте (`{id}`).  
**Опциональные параметры** могут отсутствовать, задается defaultValue.

### 3. Зачем использовать sealed class для маршрутов?
- **Типобезопасность** — компилятор проверяет существование маршрута
- **Единое хранение** — все маршруты в одном месте
- **Автодополнение** в IDE
- Предотвращает ошибки: `navController.navigate("detals/123")` (опечатка)

### 4. Что такое Back Stack и как им управлять?
**Back Stack** — стек экранов (LIFO). Для `Home → Profile → Settings`:
- Settings ← текущий
- Profile
- Home

`popBackStack()` на Settings удаляет его, возвращая на Profile.

### 5. Как работает startDestination в NavHost?
- Определяет первый экран при запуске. Можно изменить динамически:
startDestination = if (userLoggedIn) Screen.Home.route else Screen.Login.route


### 6. Что произойдёт, если навигировать на несуществующий маршрут?
- Приложение упадет с IllegalArgumentException. Решение — использовать sealed class или проверку существования маршрута

### 7. Зачем нужен параметр launchSingleTop?
Предотвращает создание дубликатов экрана в стеке. Без него 3 клика на одну дисциплину создадут 3 экрана Details в стеке. С launchSingleTop дубликаты не создаются.
